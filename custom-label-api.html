Quandosol Custom Label Printing API

Design labels inside Quandosol · Print dynamically from your own website

1. Introduction

Quandosol is a SaaS platform for designing, managing, and printing labels. This API allows customers to print labels outside of the Quandosol UI using their own applications while still using Quandosol’s label designer.

2. Eligibility & Access Control

  • Advanced Plan subscription is required
  • API key and API token must be generated from the API Credential module
  • API token is visible only once
  • Regenerating a token invalidates the old one

3. Core Concept – API Variables

Customers define API variables inside the label design instead of binding labels to items, CSVs, or API URLs. 

{IMEI}
{Model_No}
{Serial_No}
{GT_Battery_Ratio}

Values for these variables are sent at print time via apiData. Quandosol replaces them dynamically before printing.

4. High-Level Printing Flow

  1. Client sends print request
  2. API validates credentials
  3. Plan & limits are checked
  4. Label is fetched by name
  5. Variables are replaced
  6. Print records are created
  7. Final markup JSON is generated
  8. SDK renders markup
  9. Browser print dialog opens

5. API Endpoint

POST https://app.quandosol.com/api/custom-labels/print

6. Authentication Headers

HeaderDescription
X-API-KEYAPI Key
X-API-SECRETAPI Token

7. Request Body Parameters

FieldTypeRequiredDescription
label_namestringYesExact label name in Quandosol
amountintegerYesMinimum 1
apiDataobjectYesDynamic label variables

8. Example API Request

{
  "label_name": "Label A",
  "amount": 1,
  "apiData": {
    "IMEI": "356938035643809",
    "Model_No": "iPhone 13",
    "Serial_No": "SN-A12345",
    "GT_Battery_Ratio": "92%"
  }
}

9. SDK Architecture

The SDK works in two execution contexts:

  • Main page – API request & iframe trigger
  • Print page – canvas rendering & printing

11. SDK Configuration

QCIMLabelSDK.setConfig({
  apiBaseUrl: "https://app.quandosol.com/api",
  api_key: "YOUR_API_KEY",
  api_token: "YOUR_API_TOKEN",
  printPageUrl: "./print-label.html",
  debug: true
});

12. Print Trigger Example

QCIMLabelSDK.printLabel({
  label_name: "Label A",
  amount: 1,
  apiData: {
    IMEI: "356938035643809",
    Model_No: "iPhone 13"
  }
});

13. Barcode & Image Support

Supports QR Code, Code128, Code39, EAN, PDF417, DataMatrix and more. Rendered using bwip-js.

14. Browser Requirements

  • Chrome
  • Edge
  • Firefox
Printing works only in browsers due to iframe + canvas rendering.

15. CORS Policy

https://app.quandosol.com/api/custom-labels/print

16. Security Warning

Never expose API credentials in public websites. Use a backend proxy for production usage.

18. Troubleshooting

  • Ensure label_name exists
  • Ensure apiData keys match variables
  • Disable popup blockers
  • Ensure external images are publicly accessible
