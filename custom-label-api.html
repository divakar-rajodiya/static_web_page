Quandosol Custom Label Printing API

Design labels inside Quandosol · Print dynamically from your own systems

1. Introduction

Quandosol is a SaaS platform that allows businesses to design and manage label templates. This API enables printing those labels from external systems such as:

  • ERP systems
  • Warehouse software
  • Web applications
  • Desktop applications
  • POS systems

Labels are designed once in Quandosol and can then be printed dynamically using the API by sending variable values at runtime.

2. Printing Methods Overview

Method Description Best Use Case
Browser SDK Rendering inside browser using canvas Web applications
PrintNode PDF Server generates PDF and sends to printer Standard printers
PrintNode ZPL Native ZPL commands for Zebra printers Warehouse / thermal printing

3. Eligibility & Access Control

  • Advanced plan subscription required
  • API credentials must be generated
  • API tokens are visible only once
  • Regenerating token invalidates old token

4. API Variables Concept

Inside the label designer you define placeholders. Example: 
{IMEI}
{Model_No}
{Serial_No}
{GT_Battery_Ratio}
These variables will be replaced dynamically during printing. Example request: 
{
 "IMEI":"356938035643809",
 "Model_No":"iPhone 13",
 "Serial_No":"SN-A12345"
}

10. Browser Printing Example

QCIMLabelSDK.printLabel({
 label_name:"Label A",
 amount:1,
 apiData:{
  IMEI:"356938035643809",
  Model_No:"iPhone 13"
 },
 mode:"browser"
});
Browser printing works by rendering the label to canvas and triggering the browser print dialog.

11. PrintNode Printing

PrintNode allows server-side printing without browser interaction. Advantages:
  • No browser required
  • Background printing
  • Supports automation
  • Works with network printers

12. PrintNode PDF Printing

Endpoint: 
POST /api/custom-labels/print-node-pdf
Example request: 
{
 "label_name":"Label A",
 "amount":1,
 "printer_id":75236019,
 "apiData":{
  "IMEI":"356938035643809"
 }
}
Response: 
{
 "status":"sent_to_printer_png"
}

13. PrintNode ZPL Printing

Endpoint: 
POST /api/custom-labels/print-node
Example: 
{
 "label_name":"Label A",
 "amount":1,
 "printer_id":75208784,
 "apiData":{
  "IMEI":"356938035643809"
 }
}
Response: 
{
 "status":"sent_to_printer",
 "printer_dpi":203
}

ZPL Printing Architecture

Markup JSON │ ▼ Render to PNG │ ▼ PNG → ZPL (^GF) │ ▼ ZPL Commands │ ▼ Zebra Printer

ZPL Command Structure

Example ZPL: 
^XA
^PW832
^LL400
^LH0,0
^FO0,0
^GFA,...
^XZ
Explanation:
Command Description
^PWPrint width
^LLLabel length
^FOField origin
^GFGraphic field

14. Choosing the Right Printing Method

Scenario Recommended Method
Web Application Browser SDK
Office Printer PrintNode PDF
Zebra Printer PrintNode ZPL
Warehouse Automation PrintNode ZPL
Complex Graphics Labels PrintNode PDF

15. Printer Setup Guide

Steps: 1. Install PrintNode client https://www.printnode.com/download 2. Connect printer 3. Retrieve printer ID PrintNode dashboard → Printers 4. Use printer ID in API request Example: 
"printer_id":75236019

16. Error Codes

Error Description
401 Invalid API credentials
404 Label not found
422 Invalid request parameters
500 Server rendering error

17. Troubleshooting

  • Ensure label name exists
  • Check API variables match template
  • Verify printer_id is correct
  • Confirm printer is connected to PrintNode client

18. Best Practices

  • Do not expose API credentials in frontend
  • Use backend proxy for production
  • Cache label templates when possible
  • Validate API variables before sending
Quandosol enables flexible label printing from any system using browser rendering or direct server-side printing.
