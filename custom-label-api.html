Quandosol Custom Label Printing API

Design labels once inside Quandosol. Print them dynamically from any system - browser, ERP, warehouse software, or POS.

Fetch Markups Print Markups PrintNode ZPL PrintNode PDF

Introduction

Quandosol is a SaaS platform that allows businesses to design and manage label templates. This API enables printing those labels from any external system - variable data is sent at runtime, and Quandosol handles the rendering and delivery.

Compatible Systems

  • ERP systems
  • Warehouse software
  • Web applications
  • Desktop applications
  • POS systems

High-Level Flow

  • Design your label template in Quandosol
  • Define variable placeholders (e.g. {IMEI})
  • Call the API with runtime values
  • Label is rendered and sent to the printer

Methods Overview

There are four ways to interact with label data through the API. Choose the method that fits your workflow.

# Method What it Does Best Use Case Requires Printer?
1 Fetch Markups Returns raw markup JSON - no printing triggered Preview, debugging, custom rendering No
2 Print Markups Fetches markup and renders it to the browser canvas + print dialog Web applications, browser printing Browser printer
3 PrintNode ZPL Generates ZPL commands and sends directly to Zebra printer via PrintNode Warehouse / thermal printing Yes - Zebra/ZPL
4 PrintNode PDF Renders a vector PDF server-side and sends to printer via PrintNode Office / standard printers Yes - any printer

Not sure which to use? See the Choosing a Method section for a scenario-based guide.

Eligibility & Access Control

  • An Advanced plan subscription is required to access the API
  • API credentials must be generated from your Quandosol dashboard
  • API tokens are visible only once at generation time - store them securely
  • Regenerating a token immediately invalidates the previous one

API Variables Concept

Inside the label designer you define placeholder variables. These are replaced dynamically at print time with the values you send in the API request.

Define in Designer

{IMEI}
{Model_No}
{Serial_No}
{GT_Battery_Ratio}

Send at Runtime

{
  "IMEI": "356938035643809",
  "Model_No": "iPhone 13",
  "Serial_No": "SN-A12345",
  "GT_Battery_Ratio": "92%"
}

Variable names are case-sensitive. Make sure the keys in your API request exactly match the placeholders defined in the label template.

Authentication

All API requests require two authentication headers. These are available in your Quandosol dashboard under API Settings.

HeaderDescriptionExample
X-API-KEYYour API KeysplsTpbkLDzYREpXxpTLGA20V7...
X-API-SECRETYour API Tokent06utpbYol5mVgnbYp89Ym5q...

Never expose your API credentials in frontend code. In production, route API calls through a backend proxy.

SDK & Files Setup

Methods 1 and 2 (browser-based) use the Quandosol JavaScript SDK. Methods 3 and 4 (PrintNode) also use the SDK to trigger server-side printing.

1. Include the SDK

Load the SDK from Quandosol or host it yourself:

<script src="https://app.quandosol.com/qcim-label-sdk.js"></script>

2. Configure the SDK

QCIMLabelSDK.setConfig({
  apiBaseUrl:   "https://app.quandosol.com/api",
  api_key:      "YOUR_API_KEY",
  api_token:    "YOUR_API_TOKEN",
  printPageUrl: "./print-label.html",   // required for Method 2 only
  debug:        true
});

3. Create print-label.html

This file is required only for Method 2 (Print Markups). It runs inside a hidden iframe and renders the label for the browser print dialog.

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8"/>
  <title>Print Label</title>
  <style>
    html, body { margin: 0; padding: 0; background: white; }
    #printRoot  { width: 100%; height: 100%; background: white; }
    img         { display: block; }
  </style>
</head>
<body>
  <div id="printRoot"></div>
  <script src="https://unpkg.com/bwip-js/dist/bwip-js-min.js"></script>
  <script src="./qcim-label-sdk.js"></script>
</body>
</html>

Print Methods

Each method is fully independent. Use one or more depending on your needs.

1

Fetch Markups

Retrieve raw label markup JSON - no printing triggered

Preview / Debug

This method calls the API and returns the fully resolved label markup as JSON - with all variables replaced. No rendering or printing is triggered. Use this to preview label data, build custom renderers, or inspect what the label contains before committing to print.

When called via the SDK, the response is displayed in a built-in preview popup showing stage dimensions, element list, and a raw JSON view.

API Endpoint

POST /api/custom-labels/fetch-markups

Request Body

{
  "label_name": "Label A",
  "amount":     1,
  "apiData": {
    "IMEI":             "356938035643809",
    "Model_No":         "iPhone 13",
    "Serial_No":        "SN-A12345",
    "GT_Battery_Ratio": "92%",
    "Carriers":         "Unlocked"
  }
}

Response

Returns an array of markup objects. Each object contains a stage and an elements array.

[
  {
    "stage": {
      "width":      4,
      "height":     2,
      "unit":       "in",
      "background": "#ffffff"
    },
    "elements": [
      { "type": "text",    "text": "iPhone 13",        "x": 20, "y": 15, ... },
      { "type": "barcode", "format": "code128",        "text": "356938035643809", ... },
      { "type": "image",   "url": "https://...",       "x": 10, "y": 80, ... }
    ]
  }
]

SDK Usage

QCIMLabelSDK.printLabel({
  label_name: "Label A",
  amount:     1,
  apiData: {
    IMEI:             "356938035643809",
    Model_No:         "iPhone 13",
    Serial_No:        "SN-A12345",
    GT_Battery_Ratio: "92%",
    Carriers:         "Unlocked"
  },
  mode: "fetch_markups"   // <- this mode
});

Quick API Test (Browser Console)

Want to test the endpoint directly without setting up the SDK? Paste this script into your browser's DevTools console (F12 -> Console) from any regular webpage (e.g. your own site or google.com). It will hit the API and print the full response.

Do not run this from a chrome:// page - those block all outgoing requests. Use any normal http:// or https:// page instead. Also ensure your server's CORS config allows the origin you're running from (see CORS note below).

(async function () {

  const API_BASE_URL = "https://api.beta.quandosol.com/api";
  const API_KEY      = "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx";
  const API_TOKEN    = "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx";

  const LABEL_NAME = "Label A";
  const API_DATA   = {
    label_name        : "Label A",
    IMEI              : "356938035643809",
    Model_No          : "iPhone 13",
    Serial_No         : "SN-A12345",
    GT_Battery_Ratio  : "92%",
    Data_Wipe         : "Yes",
    Carriers          : "Unlocked",
    GT_Black_List     : "No",
    Functional_Results: "Pass",
    Reg_Model_Number  : "A2633",
  };
  // -----------------------------------------

  const url = `${API_BASE_URL}/custom-labels/fetch-markups`;

  console.log("%c-----------------------------------------", "color:#6366f1");
  console.log("%cQCIM fetch_markups Debug", "color:#6366f1;font-size:14px;font-weight:bold");
  console.log("%c-----------------------------------------", "color:#6366f1");
  console.log(`%c Endpoint -> %c${url}`, "color:#9ca3af", "color:#f59e0b;font-weight:bold");
  console.log("%c Payload  ->", "color:#9ca3af", { label_name: LABEL_NAME, amount: 1, apiData: API_DATA });
  console.log("%cSending request...", "color:#a78bfa;font-style:italic");

  let res, data;

  try {
    res  = await fetch(url, {
      method  : "POST",
      headers : {
        "Content-Type" : "application/json",
        "X-API-KEY"    : API_KEY,
        "X-API-SECRET" : API_TOKEN,
      },
      body: JSON.stringify({ label_name: LABEL_NAME, amount: 1, apiData: API_DATA }),
    });
    data = await res.json();
  } catch (err) {
    console.error("%cNetwork error:", "color:#f87171;font-weight:bold", err.message);
    return;
  }

  console.log(`%c HTTP ${res.status} ${res.statusText}`, res.ok ? "color:#4ade80;font-weight:bold" : "color:#f87171;font-weight:bold");

  if (!res.ok) {
    console.error("%cAPI Error:", "color:#f87171;font-weight:bold", data);
    return;
  }

  console.log("%cFull API Response:", "color:#4ade80;font-weight:bold");
  console.log(data);

})();

CORS note: If you see a CORS error in the console, your server is blocking requests from that origin. Add '*' (or the specific origin) to the allowed_origins array in your Laravel config/cors.php, then run php artisan config:clear.

2

Print Markups

Fetch markup, render to canvas, open browser print dialog

Browser SDK

This method fetches the markup (same as Method 1) and immediately renders it to an HTML canvas inside a hidden iframe. The browser's native print dialog is then triggered automatically.

This is the recommended method for web applications where no PrintNode setup is required.

Prerequisites

  • SDK loaded and configured (see SDK Setup)
  • print-label.html file present at the configured printPageUrl path
  • bwip-js loaded in print-label.html (for barcode rendering)

API Endpoint

POST /api/custom-labels/print

Request Body

{
  "label_name": "Label A",
  "amount":     1,
  "apiData": {
    "IMEI":             "356938035643809",
    "Model_No":         "iPhone 13",
    "Serial_No":        "SN-A12345",
    "GT_Battery_Ratio": "92%",
    "Carriers":         "Unlocked"
  }
}

SDK Usage

QCIMLabelSDK.printLabel({

    GT_Battery_Ratio: "92%",

3

PrintNode ZPL

Native ZPL printing to Zebra & thermal printers via PrintNode

Server-Side

This method renders the label server-side into a PNG, converts it to ZPL commands (^GF graphic field), and sends it directly to a Zebra or ZPL-compatible thermal printer via the PrintNode API - no browser required.

Prerequisites

  • PrintNode client installed and running on the machine connected to the printer
  • Printer registered in PrintNode dashboard - retrieve the printer_id
  • SDK loaded and configured

API Endpoint

POST /api/custom-labels/print-node

Request Body

{
  "label_name": "Label A",
  "amount":     1,
  "printer_id": 75208784,
  "apiData": {
    "IMEI":             "356938035643809",
    "Model_No":         "iPhone 13",
    "Serial_No":        "SN-A12345",
    "GT_Battery_Ratio": "92%",
    "Carriers":         "Unlocked"
  }
}

Response

{
  "status":      "sent_to_printer",
  "printer_dpi": 203
}

SDK Usage

QCIMLabelSDK.printLabel({
  label_name: "Label A",
  amount:     1,
  apiData: {
    IMEI:             "356938035643809",
    Model_No:         "iPhone 13",
    Serial_No:        "SN-A12345",
    GT_Battery_Ratio: "92%",
    Carriers:         "Unlocked"
  },
  mode:       "printNodeZpl",   // <- this mode
  printer_id: 75208784
});

ZPL printing is best for Zebra-brand or other ZPL-compatible thermal printers. For standard office printers, use Method 4 (PrintNode PDF) instead.

4

PrintNode PDF

Server-rendered vector PDF delivered to any printer via PrintNode

Server-Side

This method renders the label into a vector PDF server-side using TCPDF, base64-encodes it, and delivers it to any PrintNode-connected printer. It produces the highest-quality output and works with any printer that accepts PDFs.

Prerequisites

  • PrintNode client installed and running on the machine connected to the printer
  • Printer registered in PrintNode dashboard - retrieve the printer_id
  • SDK loaded and configured

API Endpoint

POST /api/custom-labels/print-node-pdf

Request Body

{
  "label_name": "Label A",
  "amount":     1,
  "printer_id": 75236019,
  "apiData": {
    "IMEI":             "356938035643809",
    "Model_No":         "iPhone 13",
    "Serial_No":        "SN-A12345",
    "GT_Battery_Ratio": "92%",
    "Carriers":         "Unlocked"
  }
}

Response

{
  "status": "sent_to_printer_png"

Printer Setup Guide

Required for Methods 3 and 4 (PrintNode). Methods 1 and 2 do not need this setup.

  1. Install the PrintNode client from https://www.printnode.com/download
  2. Launch PrintNode and connect your printer
  3. Log in to your PrintNode dashboard and navigate to Printers
  4. Copy the Printer ID for your target printer
  5. Pass the printer ID in your API request as printer_id
// Example printer IDs
"printer_id": 75236019    // Divakar's account - ZPL printer
"printer_id": 75208784    // Vinc's ZPL printer (ZDesigner ZD500R-203dpi)
"printer_id": 75208780    // Brother HL-5470DW series

Troubleshooting

  • Verify the label_name exactly matches a label saved in your Quandosol account
  • Check that all API variable keys match the placeholders defined in your label template (case-sensitive)
  • For PrintNode methods: confirm the printer_id is correct and the printer is online in the PrintNode dashboard
  • Confirm the PrintNode client is installed and running on the machine connected to the printer
  • Use Method 1 (Fetch Markups) to verify variable substitution is working correctly before attempting any print method
  • Enable debug: true in the SDK config to see detailed console logs

Best Practices

  • Never expose API credentials in frontend code. In production, proxy all API calls through your own backend.
  • Use Method 1 (Fetch Markups) during development to validate your variable data before wiring up any print method.
  • Cache label template metadata where possible to reduce repeated API calls.
  • Validate all apiData values on the client side before sending the request.
  • Store your API token securely immediately after generation - it is shown only once.

Quandosol enables flexible label printing from any system - use Fetch Markups to preview, Print Markups for browser printing, and PrintNode ZPL/PDF for direct server-side delivery to any printer.
